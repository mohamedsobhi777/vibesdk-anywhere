# Phase 3 — Preview/Deploy Lifecycle + DO-Codegen-Coupled Endpoints

> Subagent-driven (fresh implementer + reviewer per task, ledger `.superpowers/sdd/progress.md`).
> Scope from the 2-researcher Phase-3 analysis (see the ledger's PHASE 3 RESEARCH section).

**Goal:** Re-source the endpoints that read what the old Durable-Object codegen pipeline produced
(agent summary, git history, screenshots) from the **Superserve sandbox / Postgres / Supabase
Storage**, and resolve the deploy lifecycle. Most of this cannot be live-validated in this
environment (needs a live Superserve sandbox producing git/screenshots + a hosted Supabase
Storage bucket) — so, exactly like the thin vertical, deliverables are **type-correct + unit-tested
(faked Superserve/storage) + a runbook**; the live run is the user's step.

## Key decisions (made autonomously per the standing directive)
- **Deploy = the live Superserve sandbox preview (research option c).** Superserve has one primitive
  (the Sandbox), which already serves as "preview"; there is no publish/deploy primitive to promote
  into (Cloudflare's Workers-for-Platforms dispatch is gone). So retire the broken "deploy to
  Cloudflare Workers" path (it calls a stub that always fails) and make the preview URL the honest
  product surface. Real permanent-hosting (option a: keep-alive/pause-resume; option b: build→push
  to a real host) is a **deferred product decision**, not guessed at here.
- **Vault deferred.** BYOK is an optional overlay, not load-bearing (the core loop runs on the
  platform-default gateway path). The ephemeral-key session model has no serverless equivalent to
  "DO memory wiped on eviction" — a real security-posture decision, its own scoped follow-up.
- **Screenshots are optional.** Capture uses the Cloudflare Browser Rendering REST API (a hosted
  HTTPS API, not a Workers binding — runs from Bun), but gate it on credentials and skip gracefully
  when absent, so the **core loop keeps zero Cloudflare dependency**. Storage moves R2 → Supabase.

## Global constraints
Reuse: `getAgentPreviewUrl`/`Sandbox.connect` (agentSandboxBoot.ts), `AgentSessionService`, the
provider-agnostic `GitHubService` (unchanged), `fflate`/`zipExtractor.ts`, `AppService` (ported).
No `any`, no emojis, no TODO. bun. Keep root typecheck (`bun run typecheck` AND `bunx tsc -b --force`

vitest runs in directory batches (env `EADDRNOTAVAIL` ceiling ~30 files/proc). Commit per task; push
